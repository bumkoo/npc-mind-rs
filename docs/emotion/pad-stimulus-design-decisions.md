# PAD 자극 레이어 설계 결정 기록

## 문서 개요

PAD(Pleasure-Arousal-Dominance) 자극 레이어의 설계 검토 과정에서 내린
결정과 그 근거를 기록한다. 코드 수정의 이유를 추적할 수 있도록 한다.

---

## 1. PAD 레이어의 역할

### 위치: 4레이어 아키텍처에서 레이어4

```
레이어1: Situation (세계관 기준, 고정)
레이어2: HEXACO (성격, 고정)
레이어3: Relationship (상대별 관계, 대화 중 고정)
레이어4: PAD 자극 (대화 중, 매 턴 변동) ← 여기
```

### 핵심 원칙

- PAD는 **플레이어 대사의 감정적 톤**을 3차원 숫자로 요약한 것
- apply_stimulus의 **입력 형식**이며, 감정 체계 자체는 아님
- 감정은 처음부터 끝까지 OCC — PAD는 외부 자극의 형식일 뿐
- OCC → PAD 전환은 메인 파이프라인에 없음

### PAD 분석 대상

| 대사 출처 | PAD 분석 필요 | 이유 |
|---|---|---|
| 플레이어 자유 입력 | ✅ | 뭘 말할지 모름 — 엔진이 모르는 유일한 입력 |
| LLM이 생성한 NPC 대사 | ❌ | 감정이 먼저 결정되고 대사가 나옴 |
| 스크립트된 대사 | ❌ | 작가가 태깅 |

## 2. OCC→PAD 매핑 테이블

### 출처

- **Gebhard 2005 (ALMA 모델)** — 이론적 매핑. 우리 코드의 기초.
- **Hoffmann/Limbrecht 2012 (N=70 실험)** — 70명이 직접 OCC 감정을 PAD 공간에 배치한 경험적 검증.

### 주요 발견 (Hoffmann/Limbrecht 2012)

- **P축(Pleasure)**: 개인 간 일치도 높음. 표준편차 낮음. 신뢰할 수 있음.
- **A축(Arousal)**: 개인 간 편차가 P보다 큼. 그래도 방향은 일치.
- **D축(Dominance)**: 개인 간 편차가 **가장 큼**. 내적 일관성 낮음.

Shame의 D값은 두 출처 모두 -0.60 부근으로 일치.
**값이 틀린 게 아니라, pad_dot에서 쓰는 방식과 안 맞는 것.**

### 현재 매핑 테이블 (코드에 보존)

```
Anger         → P:-0.51, A:+0.59, D:+0.25
Fear          → P:-0.64, A:+0.60, D:-0.43
Shame         → P:-0.30, A:+0.10, D:-0.60
Distress      → P:-0.40, A:+0.20, D:-0.50
Joy           → P:+0.40, A:+0.20, D:+0.10
Gratitude     → P:+0.40, A:+0.20, D:-0.30
(22개 전체는 pad.rs의 emotion_to_pad 참조)
```

## 3. pad_dot D축 문제 발견

### 문제 현상

pad_dot(3축 내적)에서 D축이 복종적 감정(Shame, Fear 등)에서
부자연스러운 결과를 만든다.

### 시뮬레이션 결과

#### Shame + 비난 ❌

```
Shame(-0.30, +0.10, -0.60) · 비난(-0.6, +0.7, +0.5)
P: +0.18  A: +0.07  D: (-0.60)×(+0.5) = -0.30
합계: -0.05 → Shame 거의 안 변함

기대: 비난을 들으면 수치심이 더 커져야 함
문제: D축 반발(-0.30)이 P·A 공명(+0.25)을 상쇄
```

#### Shame + 위로 ❌

```
Shame(-0.30, +0.10, -0.60) · 위로(+0.5, -0.3, -0.4)
P: -0.15  A: -0.03  D: (-0.60)×(-0.4) = +0.24
합계: +0.06 → Shame 약간 증폭

기대: 위로를 받으면 수치심이 줄어야 함
문제: D축 공명(+0.24)이 P·A 반발(-0.18)을 상쇄
```

#### Joy (중간 문제)

```
Joy(+0.40, +0.20, +0.10) → PAD 벡터 크기 자체가 작아서
어떤 자극에도 변동 미미 (D축 문제와 별개)
```

### 원인 분석

D축은 P·A와 본질이 다르다.

- **P축, A축**: 자기 경험의 차원. "같은 방향이면 공명"이 성립.
  - 불쾌한 자극이 불쾌한 감정을 증폭 ✓
  - 고각성 자극이 고각성 감정을 증폭 ✓

- **D축**: 관계적 차원 (상보적). "같은 방향이면 공명"이 **안 성립**.
  - 지배적 자극이 복종적 감정(Shame)을 **증폭**해야 하지만, 내적은 반발로 계산
  - 복종적 자극이 복종적 감정(Shame)을 **감소**해야 하지만, 내적은 공명으로 계산

**D축의 효과가 감정마다 다르다:**

| 감정 | 지배적 자극(D:+)의 효과 | 이유 |
|---|---|---|
| Shame | **증폭** | 권위적 비난 → 더 위축 |
| Fear | **감소** | 보호자 등장 → 안심 |
| Anger | **증폭** | 상대가 당당 → 더 화남 |

단일 규칙(내적, 반전, 절대값)으로 처리 불가능.

## 4. 해결 방안 검토

### 방안 1: D축 부호 반전

```
pad_dot = P×P + A×A - D×D
```

- Shame+비난: ✓ 증폭
- Anger+도발: ❌ D축 기여가 음수 → "상대가 당당하면 화가 약해짐"

**기각**: Anger에서 부자연스러움

### 방안 2: D축 절대값

```
pad_dot = P×P + A×A + |D|×|D|
```

- Shame+비난: ✓ 증폭
- Fear+위로: ❌ D축이 공명으로 처리 → 위로 감소 효과 약화

**기각**: Fear 위로에서 부자연스러움

### 방안 3: 감정별 D반전

```
D_contribution = if emotion.D < 0 { -D×D } else { D×D }
```

- Shame+비난: ✓ 증폭
- Fear+위로: ❌ 위로(D:+)가 복종적 감정 증폭 → "보호해주는데 더 무서움"

**기각**: Fear 위로에서 부자연스러움

### 방안 4: D축 제외

```
pad_dot = P×P + A×A
```

- 모든 시나리오에서 자연스러움 ✓
- 가장 단순
- 학술 근거: D축의 내적 일관성이 P·A보다 낮음

**부분 채택**: 방향은 맞으나 D축의 영향력을 완전히 잃는 것이 아쉬움

### 방안 5: D축 차이 (|D_n - D_o|)

```
pad_dot = (P×P + A×A) + |D_n - D_o|
```

- Shame+비난: ✓ 증폭
- Shame+위로: ❌ D항이 항상 ≥0 → P·A 감소를 상쇄
- Anger+사과: ❌ D 차이가 커서 사과를 받았는데 분노 증폭

**기각**: |D| 항이 항상 양수라 감소 방향을 방해

### 방안 6 (최종 채택): D축을 P·A 결과의 스케일러로 사용

```
pad_dot = (P×P + A×A) × (1 + |D_n - D_o| × 0.3)
```

P·A가 **방향**(증폭인가 감소인가)을 결정하고,
D축 차이가 그 효과를 **얼마나 세게** 만들지를 곱한다.

직관: **상대와 나의 권력 격차가 클수록, 그 사람의 말이 나에게 더 강하게 작용한다.**

## 5. 방안 6 전수 검산

### Shame + 지배적 비난

```
Shame(-0.30, +0.10, -0.60) · 비난(-0.6, +0.7, +0.5)
P·A = +0.18 + 0.07 = +0.25 (증폭 방향)
D_scale = 1 + |(-0.60)-(+0.5)|×0.3 = 1.33
result = +0.25 × 1.33 = +0.333 → Shame 증폭 ✓
"지배적 비난이 복종적 수치심을 강하게 자극"
```

### Shame + 복종적 위로

```
Shame(-0.30, +0.10, -0.60) · 위로(+0.5, -0.3, -0.4)
P·A = -0.15 - 0.03 = -0.18 (감소 방향)
D_scale = 1 + |(-0.60)-(-0.4)|×0.3 = 1.06
result = -0.18 × 1.06 = -0.191 → Shame 감소 ✓
"비슷한 눈높이 위로 → D 차이 작아 보통 효과"
```

### Fear + 지배적 위협

```
Fear(-0.64, +0.60, -0.43) · 위협(-0.8, +0.8, +0.6)
P·A = +0.512 + 0.48 = +0.992 (증폭 방향)
D_scale = 1 + |(-0.43)-(+0.6)|×0.3 = 1.31
result = +0.992 × 1.31 = +1.30 → Fear 강한 증폭 ✓
"강한 자의 위협 → 무력한 두려움 강하게 자극"
```

### Fear + 지배적 위로

```
Fear(-0.64, +0.60, -0.43) · 위로(+0.6, -0.2, +0.5)
P·A = -0.384 - 0.12 = -0.504 (감소 방향)
D_scale = 1 + |(-0.43)-(+0.5)|×0.3 = 1.28
result = -0.504 × 1.28 = -0.645 → Fear 강한 감소 ✓
"든든한 보호자(D:+0.5)가 무력한 나(D:-0.43)를 안심 — 격차 크니 더 안심"
```

### Anger + 도발

```
Anger(-0.51, +0.59, +0.25) · 도발(-0.6, +0.7, +0.5)
P·A = +0.306 + 0.413 = +0.719 (증폭 방향)
D_scale = 1 + |(+0.25)-(+0.5)|×0.3 = 1.075
result = +0.719 × 1.075 = +0.773 → Anger 증폭 ✓
"비슷한 지배 수준 → D 효과 작음, P·A가 지배적"
```

### Anger + 사과

```
Anger(-0.51, +0.59, +0.25) · 사과(+0.5, -0.3, -0.4)
P·A = -0.255 - 0.177 = -0.432 (감소 방향)
D_scale = 1 + |(+0.25)-(-0.4)|×0.3 = 1.195
result = -0.432 × 1.195 = -0.516 → Anger 감소 ✓
"복종적 사과 → 격차 크니 효과도 큼"
```

### Joy + 축하

```
Joy(+0.40, +0.20, +0.10) · 축하(+0.7, +0.4, -0.2)
P·A = +0.28 + 0.08 = +0.36 (증폭 방향)
D_scale = 1 + |(+0.10)-(-0.2)|×0.3 = 1.09
result = +0.36 × 1.09 = +0.392 → Joy 증폭 ✓
```

### 전 방안 비교표

| 시나리오 | 기존 3축내적 | D축 제거 | D 스케일러(채택) |
|---|---|---|---|
| Shame+비난 | ❌ -0.05 | ✓ +0.25 | ✓ **+0.333** |
| Shame+위로 | ❌ +0.06 | ✓ -0.18 | ✓ **-0.191** |
| Fear+위협 | ✓ +0.734 | ✓ +0.992 | ✓ **+1.300** |
| Fear+위로 | ✓ -0.719 | ✓ -0.504 | ✓ **-0.645** |
| Anger+도발 | ✓ +0.844 | ✓ +0.719 | ✓ **+0.773** |
| Anger+사과 | ✓ -0.532 | ✓ -0.432 | ✓ **-0.516** |

## 6. 무협 시나리오 검증: 같은 P·A, 다른 D

### 같은 사과인데 상대가 다르면

교룡(Anger 0.6, D:+0.25)에게:

```
당당한 사과 (D:+0.3): "내가 잘못했소. 그건 인정하겠소."
  P·A = -0.432
  D_scale = 1 + |0.25-0.3|×0.3 = 1.015
  → Anger 감소: -0.438
  "대등한 사과 — 보통 효과"

비굴한 사과 (D:-0.5): "제...제가 잘못했습니다..."
  P·A = -0.432
  D_scale = 1 + |0.25-(-0.5)|×0.3 = 1.225
  → Anger 감소: -0.529
  "아래 사람이 고개 숙이니 화가 좀 풀린다"
```

### 같은 위로인데 상대가 다르면

수련(Shame 0.5, D:-0.60)에게:

```
사부가 위로 (D:+0.3):
  D_scale = 1 + |(-0.60)-(+0.3)|×0.3 = 1.27
  → 위로 효과 27% 증폭
  "사부가 직접 위로 → 격차 크니 더 안심"

동료가 위로 (D:0.0):
  D_scale = 1 + |(-0.60)-(0.0)|×0.3 = 1.18
  → 위로 효과 18% 증폭

후배가 위로 (D:-0.4):
  D_scale = 1 + |(-0.60)-(-0.4)|×0.3 = 1.06
  → 위로 효과 6% 증폭
  "비슷한 위치에서 위로라 보통"
```

## 7. P·A = 0 엣지 케이스 검토

### 문제

P·A 내적이 정확히 0이면 D 스케일러가 아무리 커도 결과가 0.

### 결론: 방어 코드 불필요

apply_stimulus는 **플레이어의 대사**를 분석한다.
누군가가 말을 하면 거의 항상 P 또는 A 성분이 있다.

- "물러서라" → P:-0.3, A:+0.4 (P·A ≠ 0)
- 침묵 → 대사가 없으니 apply_stimulus 자체를 호출하지 않음

"천하제일인이 다가온다"는 대사가 아니라 **상황**이다.
이건 `appraise()`가 처리한다 (SituationFocus::Event).

abs() 기반 방어 코드는 P·A의 방향 정보를 깨뜨려서 더 위험하다.

## 8. PadAnalyzer D축 임베딩

### 결정: D축 임베딩 유지

D 스케일러 방안(방안 6)을 채택하므로 PadAnalyzer에서 D축 측정을 유지한다.

- 앵커 임베딩: P 2번 + A 2번 + D 2번 = 6번 (변경 없음)
- `to_pad()`: P, A, D 모두 계산 (변경 없음)

### D축 앵커 텍스트

```
D+ (지배): "내가 주도한다, 물러서라" / "상황을 장악하고 있다" / "당당하고 자신감에 차 있다"
D- (복종): "어찌할 바를 모르겠다" / "무력하고 속수무책이다" / "위축되어 아무것도 할 수 없다"
```

## 9. 최종 pad_dot 공식

```rust
pub fn pad_dot(a: &Pad, b: &Pad) -> f32 {
    let pa = a.pleasure * b.pleasure + a.arousal * b.arousal;
    let d_gap = (a.dominance - b.dominance).abs();
    pa * (1.0 + d_gap * 0.3)
}
```

### 각 항의 역할

| 항 | 역할 | 범위 |
|---|---|---|
| `P×P + A×A` | 공명 방향 결정 (증폭/감소) | 약 -1.5 ~ +1.5 |
| `\|D_n - D_o\|` | 권력 격차 크기 | 0.0 ~ 2.0 |
| `1.0 + gap × 0.3` | 격차에 비례하는 강도 배율 | 1.0 ~ 1.6 |

### 상수

| 상수 | 값 | 역할 | 비고 |
|---|---|---|---|
| D 가중치 (W) | 0.3 | 격차가 배율에 미치는 비율 | engine.rs의 W와 동일 패턴 |
