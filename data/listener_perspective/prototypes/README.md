# 프로토타입 (분류기 기준점)

분류기가 런타임에 참조하는 발화 예시 집합.
들어온 발화와 이들 프로토타입 간 cosine 유사도로 그룹을 판정함.

## 파일

- `sign_keep.toml` — 청자 P 부호가 화자와 같은 발화 (감사, 칭찬, 비난, 위협 등)
- `sign_invert.toml` — 청자 P 부호가 화자와 반대인 발화 (사과, 간청, 위로, 빈정)

## 큐레이션 5원칙

1. **명확한 사례만.** 경계/혼합 발화는 여기 넣지 않음 (테스트 케이스 쪽으로)
2. **그룹 내 다양성.** subtype 균형 유지
3. **톤 통일.** 1인칭 대사, 무협체 (Phase 1)
4. **수집 가능성 보장.** `source` 필드로 출처 추적
5. **형식 대칭 배치.** 질문형/반복강조형은 양쪽 그룹에 의도적 대칭 배치

상세 원칙은 `docs/emotion/sign-classifier-design.md` §3.3 참조.

## 버전 관리

- `[meta].version` 값을 프로토타입 추가/수정 시마다 증분
- 벤치 결과 `run_id` 파일의 front-matter에 해당 버전이 기록됨
