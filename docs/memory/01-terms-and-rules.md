# NPC 기억 시스템 — 1차: 용어 정의 및 업무 규칙

> 본 문서는 NPC Mind Engine 기억(memory) 시스템 설계 3단계 중 **1차 산출물**이다.
> 코드 없이 도메인 용어와 업무 규칙만 정리한다. 기획·작가·엔진 개발자가 공통 언어로 논의할 수 있는 기반을 제공하는 것이 목적이다.
>
> - **1차 (본 문서)**: 업무 관점 용어·규칙
> - 2차: DDD 모델링 (애그리거트·엔티티·밸류 오브젝트·도메인 이벤트)
> - 3차: 구현 설계 (스키마·이벤트 페이로드·포트·튜닝 상수)
>
> 본 설계는 기존 `MemoryProjector` + `SqliteMemoryStore` (FTS5 + sqlite-vec vec0) 위에 **확장하는 방향**을 전제로 한다.

## 1. 개요 — 기억이 다루는 세 축

NPC 기억은 세 가지를 동시에 다룬다.

1. **세계관 지식** — NPC가 태어날 때부터 알고 있는 배경 (국가·문파·혈연). 정적이지만 사건으로 바뀔 수 있다.
2. **인물 경험** — 게임이 진행되면서 각 NPC가 겪는 사건들.
3. **타인으로부터의 정보** — 직접 겪지 않았지만 듣게 되는 것들 (전해 들음·소문).

이 셋은 저장 방식·신뢰도·수명이 달라야 자연스러운 연기가 가능하다. 같은 내용이라도 NPC가 "내가 겪었다"와 "강호에 그런 소문이 돈다"를 구분해 말할 수 있어야 한다. 이를 위해 기억은 **소유 범위(Scope)**, **출처(Source)**, **종류(Type)**, **계층(Layer)** 네 축으로 분류된다.

## 2. 용어 정의

### 2.1 기본 단위

- **기억 조각 (Memory Entry)**: 기억의 최소 단위. 한 사건·한 턴·한 사실에 대응한다. 모든 기억은 이 단위로 저장·검색·평가된다.
- **주제 (Topic)**: 여러 기억 조각이 같은 대상·사실에 대해 누적될 수 있도록 묶는 논리적 키. 예: `화산파.장문인`, `주인공.출생의비밀`. 세계관 변경이나 소문 왜곡 이력을 추적할 때 사용한다.
  - **발급 주체**: Topic은 시나리오 작가가 **선언한 식별자**다. 게임 도중 엔진이 자동 생성하지 않는다.
  - **연결 조건**: 소문 왜곡 이력·세계관 오버레이·Rumor canonical 연결은 모두 **Topic 문자열이 동일**할 때만 이뤄진다. 의미 유사도로 자동 묶이지 않는다.
  - **네이밍 규범**: 본 1차 범위에서는 규정하지 않는다 (§4 결정 유보).
- **주체 (Owner)**: 이 기억을 "누가" 소유하는가. NPC 개인일 수도, 문파·혈연·세계 전체일 수도 있다. Scope가 결정한다.

### 2.2 소유 범위 (Scope)

기억 조각은 반드시 아래 다섯 범위 중 **단 하나**에 소속한다.

- **개인 (Personal)**: 특정 NPC 한 명이 가진 기억. 대부분의 경험 기억이 여기 속한다.
- **관계 (Relationship)**: 두 NPC 사이에 공통 의미가 있는 기억. 예: "A와 B는 과거에 의형제를 맺었다."
  - **접근 모델**: 양측에서 접근 가능한 **단일 기록**이다. A와 B 모두의 회상 풀에서 잡힌다.
  - **관점 분리 원칙**: 두 NPC가 같은 사건을 근본적으로 다르게 해석하는 경우(예: "A가 B를 배신했다"를 A는 "피치 못했다"로, B는 "용서할 수 없다"로 기억)에는 Relationship Scope에 사실만 남기고, **해석·감정이 포함된 관점 기억은 각자의 Personal Scope**로 분리 저장한다.
- **문파 (Faction)**: 같은 문파 소속 NPC들이 공유하는 지식. 예: "화산파의 금기."
  - **소속 기준 시점**: Faction Scope는 **기억 생성 시점의 소속**을 기준으로 귀속된다. 이후 NPC가 문파를 이탈·전향·파문당해도 생성 당시 습득한 Faction 기억은 해당 NPC의 회상 풀에서 사라지지 않는다. 즉 "화산파에서 배운 비기"는 환속 후에도 남는다.
- **혈연 (Family)**: 같은 혈연 NPC들이 공유하는 기억. 예: 가문 비사. Family Scope도 Faction과 동일한 소속 기준 시점 규범을 따른다.
- **세계 (World)**: 이 세계의 모든 NPC가 공유할 수 있는 공적 지식. 예: "현 황제는 누구인가."

> Faction/Family/World Scope 내부의 **하위 권한 계층**(예: 문파에서 장문인만 아는 비기)은 본 1차 범위에서 다루지 않는다 (§4).

### 2.3 출처 (Source)

같은 내용의 기억이라도 **어디서 왔는가**에 따라 네 가지로 구분한다. 이 라벨이 NPC의 연기 어투와 판단 신뢰도에 직접 영향을 준다.

- **체험 (Experienced)**: 본인이 당사자로 직접 겪었다. 최고 신뢰.
- **목격 (Witnessed)**: 같은 자리에 있었으나 당사자는 아니었다. 높은 신뢰.
- **전해 들음 (Heard)**: 특정한 누군가로부터 직접 들었다. 중간 신뢰. 출처가 명확하다.
- **소문 (Rumor)**: 여러 사람을 거쳐 출처가 흐려졌다. 낮은 신뢰. 왜곡 가능.

### 2.4 종류 (Type)

기억이 **어떤 종류의 사건**에 관한 것인가. 수명·저장 조건·주입 방식이 달라지는 기준.

- **턴 기억 (Dialogue Turn)**: 특정 대화 턴에서 있었던 일. 원문에 가까운 기억.
- **장면 요약 (Scene Summary)**: 한 장면 전체의 서술적 요약.
- **장면 전환점 (Beat Transition)**: 장면 안에서 감정 흐름이 꺾인 순간.
- **관계 변화 (Relationship Change)**: 호감·신뢰·친밀도가 유의미하게 움직인 순간.
- **세계 사건 (World Event)**: 세계관 수준에서 일어난 변화.
- **문파 지식 (Faction Knowledge)**: 문파 단위 공용 지식.
- **혈연 사실 (Family Fact)**: 가문 단위 공용 기억.

> 무공 초식·의술 같은 **절차적 지식(Procedural Knowledge)**은 본 시스템 범위 밖이다. NPC 능력치 영역에서 별도 관리한다 (§4 범위 외 결정 참조).

### 2.5 계층 (Layer)

인물 경험은 두 층으로 저장된다.

- **Layer A — 구체 기억**: 강렬했던 개별 턴의 원문에 가까운 기억. 수가 많고 수명이 짧다. "그때 그 대사를 정확히 기억하는" 회상을 담당.
- **Layer B — 서술적 요약**: 장면 전체의 핵심을 한 문장으로 요약한 기억. 수가 적고 수명이 길다. "그 사건 이후 너를 의심하게 됐다"는 식의 서술적 회상을 담당.

두 층이 공존해야 구체성과 서사성이 동시에 살아난다.

**Type × Layer 기본 대응**: Type은 Layer를 함축한다. 통합(Consolidation, 규칙 3.5.5)에 의해 Layer A가 Layer B로 이동하는 경우 외에는, 생성 시점에 아래 매핑으로 결정된다.

| Type | 기본 Layer | 비고 |
|---|---|---|
| Dialogue Turn | A | 통합 대상 |
| Beat Transition | A | 통합 대상 |
| Scene Summary | B | Layer B로 직접 생성 |
| Relationship Change | B | Layer B 직접 생성, 통합 불가 (규칙 3.5.6) |
| World Event | B | Layer B 직접 생성, 통합 불가 (규칙 3.5.6) |
| Faction Knowledge | B | Layer B 직접 생성 |
| Family Fact | B | Layer B 직접 생성 |

**통합 불가 = Layer B 직접 생성 Type 중 서사적 원문 보존이 필요한 것**이라는 관계가 규칙 3.5.6의 근거다.

### 2.6 기억 동역학

- **잔존도 (Retention)**: 0~1 사이 값. 해당 기억이 현재 얼마나 또렷한가. 검색 결과 점수에 곱해지는 가중치로 작동한다.
- **망각 (Decay)**: 시간 경과에 따라 잔존도가 자연 감소하는 현상.
- **수명 상수 (Decay τ)**: 잔존도가 약 37%까지 떨어지는 데 걸리는 기간. 기억 종류·출처 조합으로 결정된다. **τ=30일은 "30일에 망각"이 아니라 "30일에 약 37% 남음"을 의미**한다.
- **회상 (Recall)**: 기억이 검색에 걸려 실제로 사용되는 사건. 회상이 일어나면 해당 기억의 잔존도 기준 시각이 갱신된다.
- **강화 (Reinforcement)**: 반복 회상으로 잔존도가 다시 오르는 효과.
- **실질 망각 한계 (Retention Cutoff)**: 잔존도가 이 값 이하로 떨어진 기억은 검색에서 제외되어 사실상 잊혀진다. 데이터는 남아 있되 NPC는 떠올리지 못한다. ※ 본 문서 내에서 "망각 한계선"이라는 표현도 동일 개념을 가리킨다.

### 2.6bis 회상 점수 요소 (Ranking Signals)

회상 시 기억의 최종 점수는 아래 다섯 요소의 조합으로 산정된다. 수식과 가중치는 3차에서 확정한다.

- **잔존도 (Retention)**: 본 기억이 현재 얼마나 또렷한가. (§2.6)
- **의미 유사도 (Semantic Similarity)**: 현재 질의(장면 상황·파트너·Focus 등)와 기억 내용이 의미적으로 얼마나 가까운가. 임베딩 코사인 유사도로 측정.
- **감정 근접도 (Emotion Proximity)**: 현재 NPC의 PAD 감정 상태와 기억이 보존하고 있는 감정 스냅샷이 얼마나 가까운가. 규칙 3.7.2의 우대 근거.
- **시간 근접도 (Temporal Recency)**: 이 기억이 생성되거나 마지막으로 회상된 시각이 현재로부터 얼마나 가까운가. 잔존도와 별개로 "최근성" 자체를 신호로 취급.
- **출처 신뢰도 (Source Confidence)**: 기억의 Source(Experienced/Witnessed/Heard/Rumor)에 따른 신뢰 가중치. Rumor/Heard의 경우 Stated Confidence × Listener Trust(§2.8)까지 반영.

### 2.7 통합과 대체

- **통합 (Consolidation)**: 흐려진 Layer A 기억 여러 개를 하나의 Layer B 요약으로 합치는 과정.
- **대체 (Supersede)**: 새로운 기억이 이전 기억을 논리적으로 무효화하거나 갱신할 때 걸리는 관계. 이전 기억은 물리적으로 삭제되지 않고 "대체됨" 표시만 받는다.
- **캐노니컬 (Canonical)**: 시나리오 시딩으로 들어온 **원본 세계관 사실**을 가리키는 상태. Canonical 기억은 감쇠하지 않고(τ=∞), 게임 도중 수정·삭제되지 않는다 (규칙 3.3.1).
- **오버레이 (Overlay)**: 세계관 진화를 표현하는 패턴. Canonical 원본은 불변으로 유지한 채, 사건에 의한 변경은 새 기억 조각으로 쌓아 올리고 이전 기억을 대체 관계로 연결한다. 원본은 불변, 변경 이력은 Topic을 통해 이어진다.

### 2.8 정보 전달과 확산

- **정보 전달 사건 (Information Telling)**: 한 NPC가 다른 NPC에게 사실 조각을 알려주는 행위. Heard 기억을 만드는 유일한 경로.
- **전달 체인 (Origin Chain)**: 이 정보가 원 출처로부터 몇 번의 입을 거쳐 왔는가. 길이가 Heard/Rumor 분기 기준.
- **소문 확산 (Rumor Spreading)**: 한 기억 조각이 지역·관계망을 따라 다수 NPC에게 동시 유입되는 사건.
- **왜곡 (Distortion)**: 소문이 전파되면서 내용 일부가 변형되는 현상.
- **도달 범위 (Reach)**: 한 번의 소문 확산이 영향을 주는 지역·문파·NPC 집합.
- **단언 강도 (Stated Confidence)**: 화자가 정보를 전달할 때 스스로 얼마나 확신하는가. Heard 기억의 신뢰도 산정에 사용.
- **청자 신뢰 (Listener Trust)**: 청자가 화자에게 가진 신뢰 수치. 같은 말도 스승에게 들으면 높게, 낯선 자에게 들으면 낮게 저장된다.
- **소문 원본 (Rumor Canonical)**: 소문의 "실제 진실"에 해당하는 기억. Rumor 조각들은 각자의 Topic을 통해 원본에 연결되거나, 원본 없이 존재하는 **고아 Rumor**일 수 있다. 원본의 유무·저장 정책은 §3.4.6에서 정한다.

### 2.9 주입 (Injection)

- **주입 경로 (Injection Path)**: 기억이 LLM에게 전달되는 경로.
- **자동 주입 (Push)**: 엔진이 장면 시작·턴·비트 전환 시점에 관련 기억을 검색하여 시스템 프롬프트에 끼워 넣는다. 정체성·세계관·상황 근접 기억이 여기 속한다.
- **능동 조회 (Pull)**: LLM이 필요할 때 스스로 회상 도구를 호출하여 기억을 끌어온다. 저신뢰 기억·특수 조회에 적합.
- **프롬프트 예산 (Prompt Budget)**: 시스템 프롬프트에 들어갈 수 있는 기억 조각 수·토큰 수의 상한.
- **기억 어투 (Memory Framing)**: 출처별로 다르게 표현되는 문장 패턴. 예: "…했다" / "…하는 것을 보았다" / "…라고 들었다" / "강호에 …라는 말이 돈다." 이 어투가 없으면 LLM이 소문을 단언해버린다.

## 3. 업무 규칙

### 3.1 기억 생성 규칙

**규칙 3.1.1 — 시딩과 이벤트 이원 경로**
모든 기억은 (a) 게임 시작 전 시나리오로부터 **시딩**되거나 (b) 게임 진행 중 **도메인 이벤트**에 의해 생성된다. 제3의 경로는 없다.

**규칙 3.1.2 — 선택적 저장**
모든 대화 턴이 기억으로 저장되지는 않는다. Layer A 턴 기억은 아래 중 하나 이상을 만족해야 저장된다.

- 감정 강도가 유의미 기준을 넘었다
- 장면 전환점(Beat Transition)이 발생했다
- 관계 수치가 유의미하게 움직였다
- 명시적 정보 전달(Information Telling)이 포함되었다

통과하지 못한 턴은 기억으로 남지 않고 이벤트 기록에만 보존된다. 유의미 기준의 구체 수치는 3차에서 정하며, 원칙적으로 기존 엔진의 `EMOTION_THRESHOLD`와 연동한다.

**규칙 3.1.2bis — 관계 변화 기억의 Scope 귀속**
"관계 수치가 유의미하게 움직였다" 조건으로 생성되는 **Relationship Change Type** 기억은 기본적으로 **Relationship Scope**에 단일 기록으로 저장된다(§2.2). 단, 두 NPC의 해석이 근본적으로 대립하는 경우에는 Relationship Scope 기록에 사실만 남기고 양측 **Personal Scope**에 해석·감정이 담긴 관점 기억을 추가로 생성할 수 있다. 추가 생성의 판정은 규칙 3.1.4와 동일한 "관점 분리" 기준을 따른다.

**규칙 3.1.3 — 장면 종료 시 요약 생성**
장면이 종료되면 반드시 Layer B 장면 요약 기억 한 건이 생성된다. 요약의 재료는 해당 장면의 Layer A 턴들이다.

**규칙 3.1.4 — 관점 분리**
한 사건에 여러 NPC가 관여했다면 기억 조각은 **관점별로 별도 생성**된다. 당사자는 Experienced, 제3자는 Witnessed로 분류된다. 하나의 이벤트가 하나의 기억으로 대표되지 않는다.

**적용 범위**: 규칙 3.1.2의 저장 조건을 통과한 턴에 대해, 해당 장면에 **존재하는 모든 참여자(당사자·제3자)별로** 관점 기억을 생성한다. "유의미하게 관여한 자만"의 선별 없이 참여자 전원에게 생성되는 것이 기본 정책이다. 장면에 존재하지 않는 NPC(부재자)에게는 생성되지 않으며, 이들이 정보를 획득하려면 반드시 정보 전달 사건(§3.4)을 경유해야 한다.

**규칙 3.1.5 — 감정 스냅샷 동반**
기억 조각은 생성 시점의 감정 PAD 스냅샷을 함께 보존한다. 이후 감정 근접 우선 회상(규칙 3.7.2)에서 사용된다.

### 3.2 기억 분류 규칙

**규칙 3.2.1 — Scope는 배타적**
하나의 기억 조각은 반드시 단 하나의 Scope에 속한다. 공유 지식을 여러 NPC가 가질 때는 Scope를 Faction/Family/World로 두어 중복 저장을 피한다.

**규칙 3.2.2 — Source는 전달 체인으로 자동 판정**
정보 전달 사건이 발생했을 때 청자 측 기억의 Source는 전달 체인 길이로 결정된다.

- 체인 길이 1 (원 출처에서 직접 들음) → **Heard**
- 체인 길이 2 이상 (여러 사람을 거침) → **Rumor**
- 출처 불명 (확산 사건으로 유입) → **Rumor**

**규칙 3.2.3 — 승격 금지, 강등 없음**
Source는 한 번 정해지면 더 신뢰도 높은 쪽으로 승격되지 않는다. Rumor로 알게 된 사실은 본인이 직접 체험하지 않는 한 Experienced가 되지 않는다. 대신 **같은 내용을 새롭게 체험하면 별도의 Experienced 기억이 추가 생성**된다.

**회상 시 우선순위**: 동일 Topic·동일 NPC에서 Source가 다른 여러 기억이 공존하는 경우(예: Rumor로 먼저 알게 된 사실을 나중에 Experienced로 재체험), 회상 시에는 기본적으로 **Source 신뢰도가 높은 쪽이 우선 선택**된다. 즉 Experienced > Witnessed > Heard > Rumor 순이며, 낮은 신뢰 버전은 같은 점수의 높은 신뢰 버전이 존재할 때 회상 풀에서 배제된다. 단 서사적으로 "두 버전을 모두 떠올리는" 연출이 필요한 경우 NPC가 Pull 회상 도구로 명시 조회할 수 있다.

**규칙 3.2.4 — Scope는 생성 시점 기준, 사후 소속 변경에 영향받지 않음**
Faction/Family Scope 기억은 생성 시점의 소속을 기준으로 귀속되며, 이후 NPC의 문파 이탈·전향·파문·환속이 있더라도 기존 기억의 접근권은 유지된다(§2.2). 신규 소속에서 얻는 기억은 새 소속 기준으로 별도 생성된다. 과거 소속 기억을 회상 풀에서 제거해야 하는 경우(예: 기억 봉인 설정)는 별도 서사 이벤트로 처리하며, 본 1차 범위 밖이다.

### 3.3 세계관 진화 규칙

**규칙 3.3.1 — 세계관 원본은 불변**
시나리오 시딩으로 들어온 세계관 기억(Canonical)은 게임 도중 수정·삭제되지 않는다.

**규칙 3.3.2 — 변경은 오버레이로만**
세계관 변경 사건은 원본을 덮어쓰지 않고 새 Overlay 기억을 쌓아 올리며 이전 기억을 대체 관계로 지정한다.

**규칙 3.3.3 — 주제별 최신 우선**
세계관 기억 검색은 같은 Topic 내에서 "대체되지 않은 가장 최신 기억"을 유효 사실로 간주한다. **"최신"의 판정 기준은 실제 시각(timestamp)이 아닌 도메인 이벤트의 시퀀스 번호**다. 본 프로젝트의 Event Sourcing 아키텍처(EventStore의 append sequence)와 정합을 맞춘다. 이로써 게임 속 시간 역행·재현·replay 시나리오에서도 결정적 결과가 보장된다.

**규칙 3.3.4 — 비동기적 인지 허용**
세계관 변경 오버레이는 자동으로 전체 NPC에게 반영되지 않는다. 해당 변경을 아는 NPC와 모르는 NPC가 같은 게임 세계에 공존할 수 있으며, 인지의 확산은 정보 전달·소문 규칙을 따른다. 이를 통해 "정보 격차로 인한 긴장"을 연출한다.

### 3.4 정보 전달 및 확산 규칙

**규칙 3.4.1 — Heard는 전달 사건에서만 생성**
Heard 기억은 명시적 정보 전달 사건(Information Telling)의 결과로만 생성된다. 대화가 있었다는 사실만으로 자동 생성되지 않는다.

**판정 경로**: 정보 전달 사건의 발행은 다음 세 경로 중 하나로 이뤄진다.

1. **(a) 명시 발행 (1차 경로)**: 시나리오 작가 또는 게임 이벤트가 `InformationTold` 커맨드/이벤트를 명시적으로 발행한다. 현재 범위에서는 **이 경로만 지원**한다.
2. **(b) 규칙 기반 탐지 (향후 확장)**: 특정 패턴·키워드·Focus 조건 충족 시 엔진이 자동 판정.
3. **(c) LLM 해석 (향후 확장)**: 턴의 대사를 LLM이 해석해 "정보 전달" 여부를 판정.

(b)(c)는 Phase 5 이후 StoryAgent·Tool 시스템 도입 단계에서 검토한다.

**규칙 3.4.2 — 엿듣기도 전달 사건**
같은 장면의 제3자가 대화를 엿들은 경우 별도의 정보 전달 사건이 발행되며, 제3자에게 Heard 기억이 생성된다. 단 신뢰도는 당사자보다 낮다.

**규칙 3.4.3 — 신뢰도 산정**
Heard 기억의 신뢰도는 화자의 **단언 강도**와 청자의 **화자에 대한 신뢰**의 곱으로 산정되며, 하한·상한이 있다. 단언 강도와 청자 신뢰는 2차/3차에서 구체 수치를 정한다.

**규칙 3.4.4 — 소문은 도달 범위 기반 일괄 유입**
소문 확산은 지역·문파·관계망으로 정의된 도달 범위 내 NPC들에게 일괄 유입된다. 범위 바깥 NPC는 해당 소문을 알지 못한다.

**규칙 3.4.5 — 확산마다 신뢰도 감소**
소문이 더 넓게 퍼질수록(전달 체인이 길수록) 신뢰도는 기하적으로 감소한다.

**규칙 3.4.6 — 왜곡은 새 기억 조각, 원본은 Canonical로 시딩**
소문이 전파되면서 내용이 변형되면 원본을 고치지 않고 왜곡된 버전을 새 기억 조각으로 만든다. 원본과는 Topic으로 연결되어 이력을 추적할 수 있다.

**원본(진실) 저장 정책**: 소문의 진실에 해당하는 **Rumor Canonical**은 다음 두 방식 중 하나로 존재한다.

- **(i) Canonical 시딩 소문**: 시나리오 작가가 해당 Topic의 "진실"을 Canonical 기억으로 시딩한 경우. 회상·검증·작가 도구에서 기준값이 된다. 권장 방식.
- **(ii) 고아 Rumor (Orphan Rumor)**: 원본 Canonical이 없는 Rumor. "누가 처음 퍼뜨렸는지 아무도 모르고 사실 여부도 시스템이 단언할 수 없는" 소문. 왜곡 버전들은 같은 Topic을 공유하며 서로의 이력을 추적하지만 진실 기준점은 없다.

고아 Rumor를 허용할지는 게임 톤의 선택이다. 본 1차는 두 방식 모두 허용한다.

**규칙 3.4.7 — 확산 주체의 범위**
소문 확산 이벤트는 (a) 시나리오 작가가 명시 발행, (b) 세계 사건의 시간차 follow-up, (c) 향후 확장의 백그라운드 틱 세 경로로 발행된다. 본 1차 문서 범위에서는 (a)와 (b)만 규정한다.

### 3.5 망각과 통합 규칙

**규칙 3.5.1 — 잔존도는 가중치, 삭제 아님**
잔존도는 기억을 삭제하지 않는다. 검색 결과의 최종 점수에 곱해지는 가중치로만 작동한다.

**규칙 3.5.2 — 수명은 종류·출처별로 다름**
수명 상수 τ는 기억 종류와 출처의 조합으로 결정된다. 체험한 장면 요약은 길게, 전해 들은 소문은 짧게 설정한다. 세계관 시딩(Canonical)은 감쇠하지 않는다.

**규칙 3.5.3 — 회상은 잔존도를 복원**
기억이 검색에 걸려 사용되면 해당 기억의 잔존도 기준 시각이 갱신되고, 누적 회상 횟수에 따라 강화 효과가 더해진다.

**규칙 3.5.4 — 실질 망각은 Retention Cutoff 필터**
잔존도가 **Retention Cutoff**(§2.6) 이하로 떨어진 기억은 검색 결과에 포함되지 않는다. 데이터는 남아 있으나 NPC는 떠올리지 못한다.

**규칙 3.5.5 — 통합으로 Layer 전환**
Layer A 기억 중 잔존도가 일정 이하로 떨어진 것들은 통합 절차를 거쳐 Layer B 요약으로 합쳐지고, 원본 Layer A 기억은 대체됨 표시를 받는다.

**규칙 3.5.6 — 통합 불가 종류**
관계 변화·세계 사건은 통합 대상이 아니다. 원문 그대로 보존한다. 통합이 서사의 핵심 근거를 희석시키는 것을 방지한다.

### 3.6 주입 규칙

**규칙 3.6.1 — 정체성은 Push**
자기 정체성, 문파·혈연 소속, 핵심 관계 기억은 장면 시작 시 시스템 프롬프트에 자동 주입된다. NPC가 "자기가 누구인지"를 망각한 채 연기하지 않도록 보장한다.

**규칙 3.6.2 — 상황 근접 기억도 Push**
현재 장면의 Focus·파트너와 의미적으로 가까운 기억 상위 소수는 장면 시작 및 Beat 전환 시 프롬프트로 주입된다.

**규칙 3.6.3 — 저신뢰 기억은 Pull**
Rumor와 오래된 Heard는 자동 주입하지 않는다. NPC가 필요하다고 판단할 때 회상 도구로 스스로 당겨오도록 한다. 이렇게 해야 소문이 연기의 배경 소재로 쓰이되 자동 사실화되지 않는다.

**규칙 3.6.4 — 출처별 어투 필수**
프롬프트에 주입되는 기억은 반드시 출처에 따라 다른 표현으로 포맷된다. 이 규칙이 없으면 NPC가 소문을 사실처럼 단언해버린다. 출처별 표현 규범은 2차/3차에서 구체 문형을 정한다.

**규칙 3.6.5 — 예산 초과 시 우선순위**
프롬프트 예산을 초과하면 다음 우선순위로 자른다.

1. 감정 근접도
2. 시간 근접도
3. 출처 신뢰도
4. 잔존도
5. 의미 유사도

### 3.7 감정 연동 규칙

**규칙 3.7.1 — 감정 스냅샷 보존**
기억 조각은 생성 시점의 감정 PAD 스냅샷을 함께 보존한다(규칙 3.1.5와 동치).

**규칙 3.7.2 — 감정 근접 우선 회상**
현재 감정 상태와 유사한 PAD 스냅샷을 가진 기억이 검색 가중치에서 우대된다. 분노 중에는 분노했던 과거 사건이 먼저 떠오른다.

**규칙 3.7.3 — 감정 점화 효과**
회상된 기억의 강한 감정이 현재 감정 상태에 약하게 영향을 줄 수 있다. 본 1차 문서에서는 가능성만 명시하고, 수식·강도는 3차에서 결정한다.

## 4. 결정 유보 항목

### 4.1 2차(DDD) 또는 3차(구현)에서 확정할 항목

- 감정 강도 cutoff, 잔존도 cutoff, τ의 구체 수치
- Topic 식별자 네이밍 규범
- 통합(Consolidation) 실행 주체 — 동기 처리 vs 비동기 배치
- 엿듣기 판정의 정확한 조건 — 거리·주의 상태·장면 규모 등
- 감정 점화 효과의 수식과 강도
- Canonical 세계관 지식의 작성 포맷 (JSON/TOML 구조)
- 프롬프트 예산 단위 — 토큰 vs 조각 수
- 출처별 기억 어투의 공식 표현 예시 (한국어·영어)
- 단언 강도와 청자 신뢰의 산식
- 회상 점수 요소(§2.6bis) 5종의 결합 수식·가중치

### 4.2 범위 외로 결정된 항목 (본 시스템이 다루지 않음)

- **절차적 지식 (Procedural Knowledge)**: 무공 초식·의술 등 습득된 기술. NPC 능력치·기능 영역에서 별도 관리한다.
- **Scope 내부 하위 권한 계층**: Faction/Family 내에서 "장문인만 아는 비기", "직계만 아는 가전" 등 하위 권한. 본 1차의 5가지 Scope가 평면 구조이므로 다루지 않는다. 필요 시 향후 확장으로 별도 검토.
- **기억 봉인·삭제**: 과거 소속의 기억을 강제로 접근 불가능하게 만드는 서사 기능. 본 Scope 기본 정책(규칙 3.2.4)과 충돌하므로 별도 서사 이벤트 기반 확장으로 미룸.

### 4.3 후속 검토 예약 항목

- **의도적 기억 숨김·거짓말**: NPC가 아는 사실을 "말하지 않거나" "일부러 왜곡해 전달하는" 연기 기능. 현재는 LLM 연기 계층의 자율성에 맡기는 것을 기본으로 하되, 엔진 레벨에서 `Stated Confidence`를 거짓/은폐 표현으로 확장할지는 Phase 5 StoryAgent 도입 단계에서 재검토.
- **정보 전달 판정의 (b)(c) 경로**: 규칙 기반 탐지 / LLM 해석 기반 자동 판정. Phase 5 이후.
- **감정 점화와 ApplyStimulus 경로 통합 방식**: 규칙 3.7.3의 감정 점화는 기존 `Command::ApplyStimulus` 파이프라인 재사용을 방향성으로 두되, 회상 이벤트를 Stimulus로 주입하는 구체 경로는 3차에서 정한다.
