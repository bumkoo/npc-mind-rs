## 7. 프롬프트 구조 개선 효과 종합

### 확인된 효과
1. **`[역할: {name}]` 통합**: 섹션 수 9→8개로 감소, 토큰 20~30개 절감. 소형 LLM에서 실질적 이점.
2. **배우 프레임**: 8턴 전체에서 지문/해설 출력 0건. 순수 대사만 출력하는 목표 100% 달성.
3. **시스템 프롬프트 갱신**: Beat 전환 시 system prompt가 새 감정·상황으로 자동 교체되며, 새 프롬프트도 `[역할: {name}]` 구조 일관 유지.
4. **LLM 연기 품질**: gemma-4 소형 모델에서도 비유 활용, 존대 유지, restriction 준수, 응답 길이 제어가 양호.

### 발견된 이슈
1. **디렉티브 행동 경향 상충**: "화를 표출하되, 상황을 지켜본다" — Beat 3(map_triumph)에서 Joy/Pride 지배임에도 이 디렉티브가 유지됨. Beat 전환 후 감정에 맞는 새 디렉티브가 생성되어야 하나, behavioral_tendency 재계산 로직 확인 필요.
2. **Hate 지배 고착**: 흑점 object의 Hate가 0.75→0.94로 계속 상승하여 Joy 0.88에도 불구하고 지배 감정 자리를 내놓지 않음. object 감정의 감쇠 또는 상쇄 로직 재검토 고려.
